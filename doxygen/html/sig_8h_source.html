25 // all signal handlers must be derived from
26 // class sig::handler. class signal will
27 // maintain a list of pointers to sig::handler
28 // objects for a signal. If a signal occurs,
29 // all sig::handler's associated with the
30 // signal are invoked.
31 
32 // sig::handler object will insert itself into
33 // the signal handler list for a signo. Its
34 // dtor will delete the signal handler object
35 // from the signal handler list for a signo.
36 // Thus if a user wants to add a signal handler,
37 // all that needs to be done is to simply
38 // instantiate the signal handler object,
39 // and if the user wants to remove the signal
40 // handler, all that needs to done is to
41 // delete the object and its dtor will remove
42 // itself from the signal handler list.
